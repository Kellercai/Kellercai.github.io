基本类型有哪几种？null是对象吗？

  1. 在JS中有6钟原始值：
  • boolean
  • null
  • undefined
  • number
  • string
  • symbol

基本类型的值是按值访问的；引用类型的值是按引用访问的。
基本类型存储的都是值，所以没有函数可以调用。'1'.toString()，之所以可以调用toString，是因为’1’这个时候已经不是基本类型了，而是被强制转换为了String类型也就是对象类型。

  1. 虽然typeof null会输出object，但是这是JS最初设计遗留至今的bug。null属于基本类型。

对象类型和基本类型的不同之处？函数参数是对象会发生什么问题吗？

  1. 基本类型存储的是值，对象类型存储的是地址（指针）。

  2. 函数参数是对象

typeof是否能正确判断类型？instanceof正确判断对象的原理是什么？

  1. typeof对于基本类型，出了null都可以显示正确的类型；typeof对于对象来说，除了函数都会显示object，所以说typeof并不难准确判断变量是什么类型。

  2. 如果需要正确判断一个对象的类型，需要用到instanceof，因为内部机制是通过原型链来判断的。

类型转换

  1. JS中类型转换只有三种情况
  • 转换为布尔值
  • 转换为数字
  • 转换为字符串

  1. 四则运算符

    加法运算符（字符串优先级最高）

  • 运算中其中一方为字符串，那么另一方需要强制转为字符串。
  • 如果一方不是字符串或者数字，那么会强制将它转换为数字或者字符串。

其他运算符（数值优先级最高）

  • 运算中只要有一方是数字，那么另一方需要强制转为数字

如何正确判断this？箭头函数的this是什么呢？

  1. this的理解

  2. 箭头函数中的this

    箭头函数其实没有this，其this的指向是包裹箭头函数的第一个普通函数的this。箭头函数使用bind，apply，call这类函数无效。

  1. 对一个函数多次bind（apply或call也一样），该函数永远取值第一个bind的对象

== 和 === 有什么区别？

  1. 对于==，如果双方的类型不一样，就会进行类型转换
    判断流程：
  • 首先判断双方的类型是否相同。类型相同的话就值比较。
  • 类型不相同的话，那么就会进行类型转换。
    1. 双方中是否有一方是null或者undefined，是的话返回true。(null == 1、undefined == 3)
    2. 双方中是否有一方是NaN，是的话返回false。(NaN == NaN // false)
    3. 判断两者类型是否为string和number，是的话将字符串转为number。(1 == '1')
    4. 其中一方是Boolean类型，将boolean类型转为number再进行判断。(('1' == true) => ('1' == 1) => (1 == 1))
    5. 其中一方是object且另一方为string、number或者symbol。将object转为基本类型再进行判断。先是调用对象的valueOf()方法，如果还是不能得到基本类型，再将其返回值调用toString()方法。该过程为ToPrimitive。('1' == { name: 'keller' } => '1' == '[object Object]')
  1. 对于===，判断两者类型和值是否相同。

什么是闭包？

函数A内部有一个函数B，函数B可以访问到函数A中的变量，那么函数B就是闭包。

什么是浅拷贝？如何实现浅拷贝？什么是深拷贝？如何实现深拷贝？

  1. 浅拷贝。
  • Object.assign

    Object.assign只会拷贝所有的属性值到新的对象中，如果属性值是对象的话，拷贝的是地址，所以并不是深拷贝。

  • 扩展运算符... 浅拷贝只能拷贝第一层，如果属性值是对象类型，还是会出现指针引用的问题。
  1. 深拷贝
  • JSON.parse(JSON.stringify(object)) 局限性：
  • 会忽略undefined
  • 会忽略symbol
  • 不能序列化函数
  • 不能解决循环引用的对象

如何理解原型？如何理解原型链？

  • Object是所有对象的爸爸，所有对象都可以通过__proto__找到它
  • Function是所有函数的爸爸，所有函数都可以通过__proto__找到它
  • 函数的prototype是一个对象
  • 对象的__proto__属性指向原型，__proto__将对象和原型连接起来组成了原型链
