事件的触发过程是怎样的？知道什么是事件代理吗？

事件流

  1. 事件捕获阶段
    • 事件从根结点流向目标节点，流经各个DOM节点，在各个节点中触发捕获事件，直到到达目标节点。
    • 捕获阶段的主要任务是建立传播路径，在冒泡阶段根据这个路径回溯到根结点。
  2. 目标阶段
    • 事件到达目标节点时，就到了目标阶段，事件在目标节点被触发，
  3. 冒泡阶段
    • 事件在目标节点上触发，并一层层往上冒，回溯到根结点。

阻止冒泡：

  • addEventListener的第三个参数useCapture默认值是false，决定注册的事件是捕获事件(true)还是冒泡事件(false)。设置为true可阻止冒泡。
  • 在事件中设置event.stopPropagation，兼容处理，需要做事件const event = 事件event || window.event

事件代理

将子节点需要注册的事件注册在父节点上。

优点：

  • 节省内存
  • 不需要给子节点注销事件

什么是跨域？为什么浏览器要使用同源策略？你有几种方式可以解决跨域问题？了解预检请求吗？

  • 协议，域名和端口有一个不同会出现跨域，ajax请求会请求失败。跨域是浏览器行为。

  • 出于安全策略，引入同源策略。主要是用来防止CSRF攻击的。简单说，CSRF攻击就是利用用户的登陆态发起恶意请求。比如用户当前处于登陆状态，如果没有同源策略，就会被任意来源的ajax访问到了信息。

  • 跨域的请求确实是发送出去了，但是被浏览器拦截了响应。

  • 跨域处理

  1. JSONP
    JSONP的原理很简单，就是利用<script>标签没有跨域限制的漏洞。通过<script>标签指向一个需要访问的地址并提供一个回调函数来接收数据。JSONP使用简单而且具有不错的兼容性，但是只限于get请求。

  2. CORS
    CORS需要浏览器和后端同时支持。浏览器会自动进行CORS通信，实现CORS通信主要是后端。服务端设置 Access-Control-Allow-Origin 就可以开启 CORS。该属性表示哪些域名可以访问资源，如果设置通配符则表示所有网站都可以访问资源。

  3. nginx 反向代理
    服务端和浏览器端约定指定的代理参数，反向代理隐藏了真正的服务器，如果浏览器访问服务器，nginx拦截指定代理参数将会将接口转发到约定的接口路径。

  4. WebSocket
    利用websocket的API，可以直接new一个socket实例，然后通过open方法内send传输到后台的值，利用message方法接收后端传来的数据。

  • Service Worker
    Service Worker是运行在浏览器背后的独立线程，一般可以用来实现缓存功能。使用Service Worker的话，传输协议必须为HTTPS。因为Service Worker中涉及到请求拦截，必须使用HTTPS协议来保障安全。

缓存策略

缓存策略都是通过设置HTTP的Header来实现的。

  • 强缓存
    强缓存可以通过两种HTTP请求头来实现：Expires和Cache-Control。强缓存表示在缓存期间不需要请求，state code为200。
    • Expires是HTTP/1的产物，受限于本地时间，如果本地时间被篡改，可能会造成缓存失效。
    • Cache-Control是HTTP/1.1的产物，优先级高于Expires。可以在请求头或者响应头中设置，并且可以组合使用多种指令。max-age
  • 协商缓存
    协商缓存可以通过两种HTTP请求头来实现：Last-Modified和Etag。如果缓存过期了，就需要发起请求验证资源是否更新。当浏览器发起请求验证资源时，如果资源没有改变，那么服务器返回304状态码，并且更新浏览器缓存有效期。
    • Last-Modified是HTTP/1的产物，表示本地文件最后修改日期
    • ETag是HTTP/1.1的产物，ETag 类似于文件指纹，If-None-Match 会将当前 ETag 发送给服务器，询问该资源 ETag 是否变动，有变动的话就将新的资源发送回来。并且 ETag 优先级比 Last-Modified 高。

重绘（Repaint）和回流（Reflow）

  • 重绘（Repaint）：当节点需要更改外观而不影响布局的，比如改变color属性。
  • 回流（Reflow）：布局或者几何属性发生改变的。

回流必定会发生重绘，重绘不一定发生回流。回流所需要的成本比重绘高得多，改变父节点里的子节点很可能导致父节点的一系列回流。

减少重绘和回流

  • 使用transform替代top
  • 使用visibility替换display: none，因为前者只会引起重绘，后者会引发回流
  • 不要把节点的属性值放在一个循环里当成循环里的变量
  • 不要使用table布局，可能很小的一个小改动都会造成table的重新布局
  • 动画实现的速度的选择，动画速度越快，回流次数越多，也可以选择使用 requestAnimationFrame
  • CSS选择符从右往左匹配查找，避免节点层级过多
  • 将频繁重绘或者回流的节点设置为图层，图层能够阻止该节点的渲染行为影响别的节点。比如对于 video 标签来说，浏览器会自动将该节点变为图层

安全防范知识点

什么是XSS攻击？如何防范XSS攻击？什么是CSP？

  • XSS简单的说就是攻击者将可执行的代码注入到网页中。XSS总体分为两种：持久型和非持久型。
  • 持久型：攻击的代码被服务器写入数据库中，导致访问页面的用户都受到攻击。比如评论功能输入可执行的代码。
  • 非持久型相比于前者危害就小的多了，一般通过修改 URL 参数的方式加入攻击代码，诱导用户访问链接从而进行攻击。

防范XSS攻击：

  • 转义字符
    转义用户输入的内容，对于引号，尖括号和斜杆进行转义。纯文本。
  • CSP
    CSP本质上就是建立白名单，开发者明确告诉浏览器哪些外部资源可以加载和执行。
    通过以下两种方式开启CSP：
  1. 设置 HTTP Header 中的 Content-Security-Policy
  2. 设置 meta 标签的方式

什么是CSRF攻击？如何防范CSRF？

CSRF：跨站请求伪造。原理就是攻击者构造一个后端请求地址，诱导用户点击或者通过某些途径自动发起请求。如果用户在登录状态下的话，后端就以为是用户在操作，从而进行相应的逻辑。

防范CSRF攻击：

  • Get请求不对数据进行修改
  • 不让第三方网站访问用户Cookie。设置SameSite，有兼容性。
  • 阻止第三方网站请求接口
  • 请求时附带验证信息，比如token或者验证码

什么是点击劫持？如何防范点击劫持？

点击劫持是一种视觉欺骗的攻击手段。攻击者将需要攻击的网站通过iframe嵌套的方式嵌入自己的网页，并将iframe设置为透明，展示一个诱导用户点击的按钮。
