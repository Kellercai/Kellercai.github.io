UDP与TCP的区别是什么？

UDP

UDP协议是面向无连接的，UDP协议只是数据报文的搬运工，不保证有序且不丢失的传递到对端，并且UDP协议也没有任何控制六楼的算法，总的来说UDP相较于TCP更加便捷。

属性：

  1. 面向无连接
  2. 不可靠性
  3. 高效
  4. 传输方式：支持一对多，多对一，多对多，也就是UDP提供了一个单播，多播，广播的功能。

适用场景：

  • 直播
  • 王者荣耀

TCP

TCP基本与UDP相反，建立连接断开连接都需要先进行握手。在传输数据的过程中，通过各种算法保证数据的可靠性，当然带来的问题就是相比UDP来说不那么高效了。

  • 建立连接需要三次握手，断开连接需要四次握手
  • 滑动窗口解决了数据的丢包、顺序不对和流量控制问题
  • 拥塞窗口实现了对流量的控制，保证在全天候环境下最优的传递数据

HTTP和HTTPS的区别？

  1. HTTP的URL以http://开头，HTTPS以https://
  2. HTTP是不安全的，HTTPS是安全的
  3. HTTP的标准端口是80，HTTPS的标准端口是443
  4. 在OSI网络模型中，HTTP工作于应用层，而HTTPS的安全传输机制工作在传输层
  5. HTTP是明文传输的，HTTPS对传输的数据进行加密
  6. HTTP无需证书，HTTPS需要CA机构颁发的TLS/SSL证书

HTTPS还是通过了HTTP来传输信息，但是信息通过了TLS/SSL协议（在应用层和传输层之间）进行加密。加密技术：对称加密和非对称加密。

  • 对称加密：对称加密就是两边都拥有相同的秘钥，两边都知道如何将密文加密解密。
  • 非对称加密：有公钥秘钥之分，公钥所有人都知道，可以将数据用公钥加密，但是将数据解密必须使用私钥解密，私钥只有分发的公钥一方才知道。

HTTP/1和HTTP/2的区别？

同域名请求并发数量限制？

  • HTTP/1为了性能考虑需要引入雪碧图，将小图内联，使用多个域名等等方式。这一切都是因为浏览器限制了同一个域名下的请求数量（Chrome一般是限制为6个）。当页面请求很多资源的时候，就会出现队头阻塞，导致剩下的资源需要等其他资源请求完成后才能发起请求。
  • HTTP/2引入了多路复用的技术，这个技术可以只通过一个TCP连接就可以传输所有的请求数据。

二进制传输

HTTP/1都是通过文本的方式传输数据。HTTP/2引入了新的编码机制，所有传输的数据都会被分割，并采用二进制格式编码。

Header 压缩

  • 在 HTTP/1 中，我们使用文本的形式传输 header，在 header 携带 cookie 的情况下，可能每次都需要重复传输几百到几千的字节。
  • 在 HTTP/2 中，使用了 HPACK 压缩格式对传输的 header 进行编码，减少了 header 的大小。并在两端维护了索引表，用于记录出现过的 header ，后面在传输过程中就可以传输已经记录过的 header 的键名，对端收到数据后就可以通过键名找到对应的值。
